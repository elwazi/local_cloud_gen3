## Project Overview

Infrastructure-as-code project for deploying eLwazi/GeneMap Gen3 ‚Äî a private genomic data management platform on OpenStack. The stack is Packer (image building) ‚Üí Terraform (cloud provisioning) ‚Üí Ansible (service configuration).

## Common Commands

### Setup (once)

```bash
# Source OpenStack credentials before any cloud operations
source eLwazi-openrc.sh

# Install Python dependencies (ansible, openstackclient, s3cmd, s3fs)
uv sync

# Initialize Packer plugins
packer init images.openstack.pkr.hcl

# Initialize Terraform
terraform init

# Copy and populate configuration
cp variables.auto.hcl.template variables.auto.hcl
cp group_vars/all.template group_vars/all
```

### Build & Deploy

```bash
# Build OpenStack VM images (idempotent ‚Äî skips existing images)
./build.sh

# Provision cloud infrastructure
terraform plan
terraform apply

# Configure all infrastructure (runs playbooks in parallel where safe)
./deploy.sh

# Run a single Ansible playbook
ansible-playbook rancher.yml
ansible-playbook gen3.yml

# Run a specific Ansible role/tag
ansible-playbook site.yml --tags database
ansible-playbook site.yml --limit rancher_servers
```

## Architecture

### Deployment Pipeline

1. **`variables.auto.hcl`** ‚Äî secrets/config values for local use; symlinked as `variables.auto.pkrvars.hcl` (Packer) and `variables.auto.tfvars` (Terraform)
   **`variables.hcl`** ‚Äî variable definitions shared by Packer (`variables.pkr.hcl`) and Terraform (`variables.tf`) via symlinks
2. **Packer** (`images.openstack.pkr.hcl`) ‚Äî builds base, rancher-server, rancher-worker, load-balancer, and database images from Ubuntu 24.04
3. **Terraform** (`network.tf`, `nodes.tf`, `resources.tf`) ‚Äî provisions OpenStack network, security groups, VMs, and volumes; outputs a generated `inventory.yaml` via `templates/inventory.yaml.tpl`
4. **Ansible** ‚Äî configures VMs using the generated inventory; `deploy.sh` orchestrates parallel execution

### Ansible Playbook Ordering

`deploy.sh` enforces this ordering:
1. `common.yml` ‚Äî sequential, must finish first
2. `database.yml`, `storage.yml`, `load_balancer.yml`, `rancher.yml` ‚Äî run in parallel
3. `argocd.yml` ‚Äî sequential; installs ArgoCD on the gen3 host
4. `gen3.yml` ‚Äî sequential, runs last; deploys Gen3 services via ArgoCD/Helm

`site.yml` imports all sub-playbooks sequentially for manual full-stack runs.

### Ansible Roles

| Role | Purpose |
|------|---------|
| `base` | OS hardening, packages, timezone |
| `common` | Shared services across all nodes |
| `database` | PostgreSQL with 17+ Gen3-specific users/DBs |
| `rancher` | RKE2 Kubernetes cluster (server + workers) |
| `storage` | Garage S3-compatible object storage |
| `load_balancer` | Nginx reverse proxy |
| `argocd` | ArgoCD GitOps controller on the gen3 host |
| `gen3` | Helm chart deployment of Gen3 microservices |

### Other Playbooks

- `base.yml` ‚Äî runs `base` role against `base_image` hosts (used during Packer image builds)
- `docker.yml` ‚Äî installs Docker on `docker_image`/`docker_nodes` hosts (Packer image builds)

### Key Files

- `variables.auto.hcl` ‚Äî local secrets/config (not committed; based on `variables.auto.hcl.template`)
- `group_vars/all` ‚Äî Ansible variables (not committed; based on `group_vars/all.template`)
- `inventory.yaml` ‚Äî generated by Terraform; do not edit manually
- `terraform.tfstate` ‚Äî tracked in git (intentional for this project)
- `ansible.cfg` ‚Äî sets Python 3.12 interpreter, 8 forks, 2-hour fact cache, smart gathering
- `pyproject.toml` ‚Äî Python project deps (ansible, python-openstackclient, s3cmd, s3fs); install with `uv sync`
- `gitops/gen3/values.yaml` ‚Äî Helm values file applied by ArgoCD for Gen3 services

## Important Notes

- Always `source eLwazi-openrc.sh` before running `./build.sh`, Terraform, or Ansible commands
- `./build.sh` checks for existing images in OpenStack before building ‚Äî safe to re-run
- Parallel jobs in `deploy.sh` write logs to `logs/` directory
- `packer init` only needs to be run once to install plugins
