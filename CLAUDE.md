## Project Overview

Infrastructure-as-code project for deploying eLwazi/GeneMap Gen3 — a private genomic data management platform on OpenStack. The stack is Packer (image building) → Terraform (cloud provisioning) → Ansible (service configuration).

## Common Commands

### Setup (once)

```bash
# Source OpenStack credentials before any cloud operations
source eLwazi-openrc.sh

# Install Python dependencies (ansible, openstackclient, s3cmd, s3fs)
uv sync

# Initialize Packer plugins
packer init images.openstack.pkr.hcl

# Initialize Terraform
terraform init

# Copy and populate configuration
cp variables.auto.hcl.template variables.auto.hcl
cp group_vars/all.template group_vars/all
```

### Build & Deploy

```bash
# Build OpenStack VM images (idempotent — skips existing images)
./build.sh

# Provision cloud infrastructure
terraform plan
terraform apply

# Configure all infrastructure (runs playbooks in parallel where safe)
./deploy.sh

# Run a single Ansible playbook
ansible-playbook rancher.yml
ansible-playbook gen3.yml

# Run a specific Ansible role/tag
ansible-playbook site.yml --tags database
ansible-playbook site.yml --limit rancher_servers
```

## Architecture

### Deployment Pipeline

1. **`variables.auto.hcl`** — secrets/config values for local use; symlinked as `variables.auto.pkrvars.hcl` (Packer) and `variables.auto.tfvars` (Terraform)
   **`variables.hcl`** — variable definitions shared by Packer (`variables.pkr.hcl`) and Terraform (`variables.tf`) via symlinks
2. **Packer** (`images.openstack.pkr.hcl`) — builds base, rancher-server, rancher-worker, and load-balancer images from Ubuntu 24.04 (database image build is disabled; database node boots from the load-balancer image)
3. **Terraform** (`network.tf`, `nodes.tf`, `resources.tf`) — provisions OpenStack network, security groups, VMs, and volumes; outputs a generated `inventory.yaml` via `templates/inventory.yaml.tpl`
4. **Ansible** — configures VMs using the generated inventory; `deploy.sh` orchestrates parallel execution

### Ansible Playbook Ordering

`deploy.sh` enforces this ordering:
1. `common.yml` — sequential, must finish first
2. `database.yml`, `storage.yml`, `load_balancer.yml`, `rancher.yml` — run in parallel
3. `argocd.yml` — sequential; installs ArgoCD on the gen3 host
4. `gen3.yml` — sequential, runs last; deploys Gen3 services via ArgoCD/Helm

`site.yml` imports all sub-playbooks sequentially for manual full-stack runs.

### Ansible Roles

| Role | Purpose |
|------|---------|
| `base` | OS hardening, packages, timezone |
| `common` | Shared services across all nodes |
| `database` | PostgreSQL with 17+ Gen3-specific users/DBs |
| `rancher` | RKE2 Kubernetes cluster (server + workers) |
| `storage` | Garage S3-compatible object storage |
| `load_balancer` | Caddy reverse proxy |
| `argocd` | ArgoCD GitOps controller on the gen3 host |
| `gen3` | Helm chart deployment of Gen3 microservices |

### Other Playbooks

- `base.yml` — runs `base` role against `base_image` hosts (used during Packer image builds)
- `docker.yml` — installs Docker on `docker_image`/`docker_nodes` hosts (Packer image builds)

### Key Files

- `variables.auto.hcl` — local secrets/config (not committed; based on `variables.auto.hcl.template`)
- `group_vars/all` — Ansible variables (not committed; based on `group_vars/all.template`)
- `inventory.yaml` — generated by Terraform; do not edit manually
- `terraform.tfstate` — tracked in git (intentional for this project)
- `ansible.cfg` — sets Python 3.12 interpreter, 8 forks, 2-hour fact cache, smart gathering
- `pyproject.toml` — Python project deps (ansible, python-openstackclient, s3cmd, s3fs); install with `uv sync`
- `gitops/gen3/values.yaml` — Helm values file applied by ArgoCD for Gen3 services

## Important Notes

- Always `source eLwazi-openrc.sh` before running `./build.sh`, Terraform, or Ansible commands
- `./build.sh` checks for existing images in OpenStack before building — safe to re-run
- Parallel jobs in `deploy.sh` write logs to `logs/` directory
- `packer init` only needs to be run once to install plugins
