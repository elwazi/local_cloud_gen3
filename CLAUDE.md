# CLAUDE.md

This file provides guidance to Claude Code when working with this repository.

## Project Overview

Infrastructure-as-code for deploying a private Gen3 genomic data platform on OpenStack. Pipeline: Packer (image building) → Terraform (cloud provisioning) → Ansible (service configuration). Gen3 services run on RKE2 Kubernetes managed by ArgoCD.

## Key Commands

```bash
source eLwazi-openrc.sh      # required before any cloud operations
./build.sh                   # build OpenStack VM images (idempotent)
terraform apply              # provision infrastructure → generates inventory.yaml
./deploy.sh                  # full Ansible deployment (first time)
./update.sh                  # routine updates: re-upload user.yaml + re-apply ArgoCD Application
ansible-playbook gen3.yml --tags argocd  # Helm values / portal only
ansible-playbook gen3.yml --tags user    # user.yaml only
```

## Configuration Files

### `variables.auto.hcl` — Packer + Terraform secrets/config
Not committed. Copied from `variables.auto.hcl.template`. Symlinked as `variables.auto.pkrvars.hcl` and `variables.auto.tfvars`. Contains: OpenStack credentials, node flavours/sizes, Gen3 hostname, Google OAuth, PostgreSQL and Garage S3 credentials.

### `group_vars/all` — Ansible variables
Not committed. Copied from `group_vars/all.template`. Two key sections:

**`gen3_users`** — list of identities with platform access; rendered into `user.yaml` and uploaded to Garage S3 for fence/usersync:
```yaml
gen3_users:
  - email: user@example.com
    name: User Name
    groups: [data_submitters, data_readers, indexd_admins]
    policies: [eLwaziProject_submitter]
```

**`gen3_portal`** — portal branding and customisation:
```yaml
gen3_portal:
  appName: "..."
  nav_title: "..."
  heading: "..."
  intro_text: "..."
  login_title: "..."
  login_subtitle: "..."
  login_text: "..."
  logo_file: my_logo.png        # filename in roles/gen3/files/portal/
  css: |
    /* injected into portal as gitops.css */
  sponsors:
    - file: sponsor.svg
      href: https://example.org
      alt: Sponsor
```

### `inventory.yaml` — generated by Terraform
Do not edit manually. Re-generated on every `terraform apply`.

## Architecture

### Deployment pipeline

1. `variables.auto.hcl` feeds both Packer and Terraform via symlinks
2. Terraform provisions OpenStack network, security groups, VMs, and volumes; emits `inventory.yaml`
3. `deploy.sh` runs Ansible in order: `common.yml` → parallel(`database.yml`, `storage.yml`, `load_balancer.yml`, `rancher.yml`) → `argocd.yml` → `gen3.yml`

### Gen3 deployment model (push, not pull)

Ansible renders `roles/gen3/templates/argocd-application.yaml.j2` with all Helm values inline and pushes it to the cluster via `kubectl apply`. ArgoCD then pulls the Gen3 Helm chart from `https://helm.gen3.org` and deploys. **There is no GitOps pull from GitHub** — Ansible is the source of truth for configuration.

### Ansible roles

| Role | Purpose |
|------|---------|
| `base` | OS hardening, packages, timezone |
| `common` | Shared services across all nodes |
| `database` | PostgreSQL with Gen3-specific users/DBs |
| `rancher` | RKE2 Kubernetes cluster (server + workers) |
| `storage` | Garage S3-compatible object storage |
| `load_balancer` | Caddy reverse proxy |
| `argocd` | ArgoCD on the gen3 host |
| `gen3` | Uploads `user.yaml` to S3; applies the ArgoCD Application |

### Portal branding

Image assets live in `roles/gen3/files/portal/`. Ansible slurps them at deploy time and injects them as base64 data URIs. To change an asset: replace the file and run `./update.sh --tags argocd`. Text and CSS come from `gen3_portal` in `group_vars/all`.

## Important Notes

- `terraform.tfstate` is gitignored (like `variables.auto.hcl` and `inventory.yaml` — all secrets/state stay off-repo)
- `ansible.cfg` uses `result_format=yaml` and `profile_tasks` for readable output; logs to `logs/ansible.log`
- Parallel jobs in `deploy.sh` write per-playbook logs to `logs/`
- `group_vars/all.template` is the canonical reference for all Ansible variables
